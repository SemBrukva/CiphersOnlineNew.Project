# Правила мультиязычных URL

Документ описывает поведение URL при включённой мультиязычности (`APP_MULTILANG=true`).

---

## Структура URL

| Сценарий | Формат URL |
|----------|-----------|
| Дефолтный язык (`APP_LOCALE`) | `/page` — без префикса |
| Прочие языки | `/{lng}/page` |
| Приватные страницы (только авторизованным) | `/page` — никогда без языкового префикса |

---

## Гость

- Имеет доступ только к публичным страницам.
- Язык определяется по URL-префиксу: `/ru/contacts` → locale `ru`.
- Дефолтный язык работает без префикса: `/contacts` → locale из `APP_LOCALE`.
- Все ссылки в меню, навигации и шаблонах генерируются с тем же префиксом, что и текущая страница. Смена языка — только через переключатель языков.

---

## Авторизованный пользователь

Язык пользователя хранится в `users.language` и является единственным источником истины. URL не влияет на применяемый язык.

### Приватные страницы (доступны только авторизованным)

Признак: маршрут содержит `AuthMiddleware` в своём middleware-стеке.

| URL | Поведение |
|-----|-----------|
| `/cabinet` | ✅ Нормальный доступ |
| `/ru/cabinet` | ❌ 404 |

Языковой префикс в приватных URL не допускается.

### Публичные страницы

URL должен содержать языковой префикс, соответствующий `users.language`. При несоответствии выполняется `301`-редирект.

| `users.language` | Запрошен URL | Поведение |
|-----------------|-------------|-----------|
| `ru` | `/contacts` | 301 → `/ru/contacts` |
| `ru` | `/ru/contacts` | ✅ Нормальный доступ |
| `ru` | `/en/contacts` (не-дефолт.яз.) | 301 → `/ru/contacts` |
| `en` (дефолт) | `/ru/contacts` | 301 → `/contacts` |
| `en` (дефолт) | `/contacts` | ✅ Нормальный доступ |
| `ru` | `/` (главная) | 301 → `/ru` |
| `ru` | `/ru` (главная) | ✅ Нормальный доступ |

### Логин

Авторизация выполняется через `POST /api/auth/login`. После успешного входа приложение перенаправляет пользователя на главную страницу. `LocaleMiddleware` на следующем запросе автоматически скорректирует URL под `users.language`, независимо от того, с какой языковой версии страницы пользователь входил.

### Смена языка

Управляется переменной окружения `USER_CAN_CHANGE_LANGUAGE`.

- `false` (по умолчанию) — язык пользователя неизменяем.
- `true` — пользователь может сменить язык в личном кабинете через обновление поля `users.language`.

При смене языка пересчёт никаких данных не требуется — язык влияет только на отображение.

### Регистрация

Язык нового пользователя (`users.language`) устанавливается по языковому префиксу URL страницы регистрации:
- `/registration` → `APP_LOCALE` (дефолт)
- `/ru/registration` → `ru`

---

## Реализация

### `LocaleMiddleware`

Глобальный middleware. Порядок применения:

1. Извлечь языковой префикс из URL → `$detectedLocale`, `$strippedPath`, `$hasPrefix`.
2. Определить тип посетителя:
   - **Гость** — установить `translator.locale` по `$detectedLocale`, передать роутеру `$strippedPath`.
   - **Авторизованный** — установить `translator.locale` по `users.language`, затем:
     - Если маршрут **приватный** или **не найден** в веб-таблице (например, `/admin/...`):
       - Если `$hasPrefix` → вернуть `404`.
       - Иначе → передать управление дальше.
     - Если маршрут **публичный**:
       - Вычислить ожидаемый URL (`buildLocalePath()`).
       - Если текущий URL ≠ ожидаемому → `301`-редирект.
       - Иначе → передать роутеру `$strippedPath`.

### `RouteMatcherInterface`

Интерфейс `App\Http\RouteMatcherInterface` реализован `Router`. Позволяет `LocaleMiddleware` проверять тип маршрута до его диспетчеризации, не создавая циклической зависимости.

### `locale_url()`

Хелпер генерирует URL с правильным языковым префиксом на основе текущего `translator.locale` (для гостей — из URL; для авторизованных — из `users.language`).

---

## Переменные окружения

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|----------|
| `APP_MULTILANG` | `false` | Включить мультиязычность |
| `APP_LOCALE` | `en` | Дефолтный язык (без URL-префикса) |
| `APP_LANGUAGES` | `en` | Все поддерживаемые языки через запятую |
| `USER_CAN_CHANGE_LANGUAGE` | `false` | Разрешить пользователю менять язык |
